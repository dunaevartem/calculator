Данное веб-приложение написанное на языке golang с помощью написанного пайплайна gitlab-ci.yml билдится, тестируется (main_test.go если добавить в папку test другие тесты то они поочередно будут выполняться) и деплоится на физ./вирт. сервер в виде службы через написанный ansible-playbook (deploy.yml).\
---
Для корректной отработки пайплайна необходимо осуществить ssh-соединение между раннером и сервером для деплоя, этот ssh-ключ будет скопирован и отправлен с раннера в созданный им контейнер докера.\
---
В файле hosts можно перечислить все серверы на который требуется установить данное веб приложение.\
---
Файл go.mod — это описание проекта (какие зависимости нужно использовать, их версии, и какую версию использует Gin). Файл go.sum — это хеш-суммы зависимостей (чтобы проверить целостность файлов и предотвратить подделку, хранит хеш-суммы для Gin).\
---
Файл myapp.service создан для поднятия веб-приложения в виде службы.
